# 网文写作新手引导 - 从零到发布

> 本指南面向完全新手，手把手教学。每一步都有具体命令和预期输出。
>
> **预计耗时**：初次 setup 约30分钟，第一章写作约20分钟。

---

## 第0章：环境准备

### 0.1 安装 Python 3.9+

1. 访问 https://www.python.org/downloads/
2. 下载 Python 3.9 或更高版本（推荐 3.10/3.11）
3. 安装时**勾选** `Add Python to PATH`

**验证**：
```bash
python --version
# 输出：Python 3.x.x
```

### 0.2 安装 OpenCode

1. 访问 https://opencode.ai/download
2. 下载并安装 OpenCode
3. 启动 OpenCode

### 0.3 获取 API Key

本项目需要两个 API 服务：

#### Embedding API（用于语义检索）

1. 访问 https://www.modelscope.cn/
2. 注册账号
3. 创建 API Key：
   - 个人中心 → API Key 管理 → 创建
4. 复制 Key（后续使用）

#### Rerank API（用于检索结果重排）

1. 访问 https://jina.ai/
2. 注册账号
3. 获取 API Key
4. 复制 Key

## 第1章：项目初始化

### 1.1 运行安装脚本

在 OpenCode 中打开终端：

```bash
cd 你的项目目录
python install.py
```

**预期输出**：
```
正在安装 Webnovel Writer...
依赖已安装
创建配置目录...
完成！
```

### 1.2 配置 .env 文件

在项目根目录（与 .opencode 同级）创建 `.env` 文件：

```bash
# Embedding 模型配置
EMBED_BASE_URL=https://api-inference.modelscope.cn/v1
EMBED_MODEL=Qwen/Qwen3-Embedding-8B
EMBED_API_KEY=你的ModelScope_API_Key

# Rerank 模型配置
RERANK_BASE_URL=https://api.jina.ai/v1
RERANK_MODEL=jina-reranker-v3
RERANK_API_KEY=你的Jina_API_Key
```

**注意**：
- `EMBED_API_KEY` 和 `RERANK_API_KEY` 替换为你在 0.3 获取的真实 Key
- 不要有空格在 `=` 两侧

### 1.3 初始化项目

在 OpenCode 中输入命令启动初始化：

```
/webnovel-init
```

系统会依次提问：

| 问题 | 示例回答 | 说明 |
|------|----------|------|
| 书名 | 《逆天修仙传》 | 你的小说名字 |
| 题材 | 玄幻 | 仙侠/都市/宫斗等 |
| 金手指 | 随身戒指带老爷爷 | 主角的特殊能力 |
| 目标字数 | 100万 | 计划总字数 |
| 更新频率 | 日更3000 | 每天计划写作字数 |

**回答所有问题后**，项目结构会自动生成：

```
你的项目/
├── .webnovel/          # 状态管理
│   ├── state.json      # 小说状态
│   └── index.db        # 向量索引
├── 正文/               # 章节正文
│   └── 第1卷/
├── 设定集/             # 角色/势力/物品设定
├── 大纲/               # 卷纲/章纲
└── .env               # API配置
```

## 第2章：规划大纲

### 2.1 生成总纲

输入命令：

```
/webnovel-plan
```

系统会依次询问：

| 问题 | 说明 |
|------|------|
| 卷数 | 计划写几卷 |
| 核心冲突 | 故事的主要矛盾 |

**回答后**，会在 `大纲/` 目录生成：
- `总纲.md` - 整本书的故事主线
- `卷1纲.md` - 第1卷的详细规划

### 2.2 查看大纲

生成的大纲文件在 `大纲/` 目录，可以用命令查看：

```bash
cat 大纲/总纲.md
```

## 第3章：开始写作

### 3.1 写第1章

输入命令：

```
/webnovel-write --chapter 1
```

**写作流程**（自动完成）：

```
Step 0: 预检 ✓
Step 0.5: 工作流记录 ✓
Step 1.5: 债务检查（首次无债务）✓
Step 1: 搜集上下���（Context Agent）✓
Step 2A: 生成正文 ✓
Step 3: 审查（多个检查器）✓
Step 4: 润色 ✓
Step 5: 存盘 ✓
Step 6: 更新状态 ✓
```

### 3.2 查看生成的章节

章节文件生成在 `正文/` 目录：

```bash
cat 正文/第1卷/第0001章-标题.md
```

### 3.3 债务标签使用（重要！）

在正文中使用标签创建伏笔和偿还：

#### 创建伏笔

```markdown
主角意外发现了[伏笔:神秘玉佩]，玉佩中似乎封印着一道残魂。
```

#### 偿还伏笔

```markdown
经过努力，主角终于将[回收:神秘玉佩]中的残魂唤醒。
```

**债务规则**：
- `[伏笔:xxx]` - 自动创建 HIGH 优先级债务
- `[回收:xxx]` - 自动标记债务已偿还
- 高优先级债务存在时，无法写高潮章节

## 第4章：质量审查

### 4.1 手动审查

如果想单独审查某章节：

```
/webnovel-review --chapter 1
```

### 4.2 审查报告解读

审查结果包含以下字段：

| 字段 | 含义 | 分数范围 |
|------|------|----------|
| `overall_score` | 综合评分 | 0-100 |
| `pass` | 是否通过 | true/false |
| `severity_counts` | 问题严重程度统计 | critical/high/medium/low |

**分数参考**：
- 90-100：优秀
- 80-89：良好
- 70-79：合格
- 60-69：需要修改
- <60：问题较多

### 4.3 问题严重程度

| 级别 | 说明 | 是否阻断 |
|------|------|----------|
| **critical** | 致命问题（如战力崩坏） | 阻断写作 |
| **high** | 严重问题（如OOC） | 建议修改 |
| **medium** | 中等问题 | 可选修改 |
| **low** | 轻微问题 | 忽略即可 |

### 4.4 债务检查原理

 DebtTracker 债务追踪系统：
- **HIGH 优先级债务** → 硬阻断高潮章节
- **>3 个活跃债务** → 高潮章节阻断，普通章节只警告

## 第5章：批量写作

### 5.1 连续写多章

一次性写多章（建议从第2章开始）：

```
/webnovel-write-batch --range 2-10
```

**参数说明**：

| 参数 | 说明 | 示例 |
|------|------|------|
| `--range` | 章节范围 | `2-10` |
| `--review-level` | 审查级别 | `minimal`/`standard`/`full` |

**审查级别**：

| 级别 | 说明 | 速度 |
|------|------|------|
| `minimal` | 只运行核心审查器 | 最快 |
| `standard` | 核心+条件审查器 | 中等 |
| `full` | 全部审查器 | 最慢 |

### 5.2 断点恢复

如果批量写作中断，可以使用恢复命令：

```
/webnovel-resume
```

系统会检测中断位置，让你选择是否继续。

## 第6章：导出与发布

### 6.1 导出正文

将正文导出为不同格式：

```bash
# 导出为 Markdown
/webnovel-export --format markdown

# 导出为 TXT
/webnovel-export --format txt

# 导出为 EPUB（电子书）
/webnovel-export --format epub
```

导出文件保存在 `导出/` 目录。

### 6.2 发布到番茄小说

**首次发布**（需要配置）：

```
/webnovel-publish
```

按照交互提示完成：

| 步骤 | 操作 |
|------|------|
| 1 | 安装 Playwright（首次） |
| 2 | 登录番茄作家后台 |
| 3 | 创建/选择书籍 |
| 4 | 选择上传的章节 |
| 5 | 确认发布 |

**后续发布**：

只需输入 `/webnovel-publish`，系统会记住配置，直接进入第4步。

## 第8章：常见问题FAQ

### Q1: API Key 从哪获取？

详见第0章「环境准备」：
- Embedding Key：ModelScope (modelscope.cn)
- Rerank Key：Jina AI (jina.ai)

### Q2: 写一半卡文了怎么办？

1. 使用 `/webnovel-query` 查询角色或物品设定
2. 使用 `/webnovel-dashboard` 可视化查看剧情进度
3. 查看 `大纲/` 目录下的规划

### Q3: 债务怎么偿还？

在正文中使用 `[回收:伏笔名]` 标签即可偿还。

```markdown
[回收:神秘玉佩]
```

### Q4: 如何切换题材？

编辑 `.env` 文件或项目配置：

```bash
# 可选题材：xianxia, urban, scifi, fantasy
world_preset=xianxia
```

### Q5: 审查分数太低怎么办？

1. 先看审查报告中的 `severity_counts`
2. 优先修复 `critical` 和 `high` 问题
3. 使用 `/webnovel-review --chapter X` 重新审查确认

### Q6: 如何查看项目状态？

```bash
/webnovel-dashboard
# 或
python .opencode/scripts/webnovel.py status
```

## 附录：债务系统详解

### 债务类型

| 类型 | 创建方式 | 优先级 |
|------|----------|--------|
| 显式伏笔 | `[伏笔:xxx]` | HIGH |

| 条件 | 普通章节 | 高潮章节 |
|------|----------|----------|
| HIGH priority 债务存在 | 警告 | **阻断** |
| >3 活跃债务 | 警告 | **阻断** |
