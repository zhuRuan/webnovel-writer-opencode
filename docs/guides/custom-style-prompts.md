# 自定义文风指南

> 用你自己的话告诉 AI 怎么写。

## 快速开始

在你的书项目目录下创建 `设定集/prompts/` 文件夹，放入 `.md` 文件即可。系统每章写作时会自动读取并注入。

```
你的书/
├── 设定集/
│   ├── 世界观.md
│   ├── 力量体系.md
│   └── prompts/          ← 在这里放你的提示词
│       ├── 文风.md
│       └── 对话风格.md
├── 正文/
├── 大纲/
└── .story-system/
```

## 文件命名

- 文件名随意，按字母排序加载
- 建议用有意义的名字：`文风.md`、`对话.md`、`禁忌.md`、`节奏.md`
- 可以有多个文件，也可以只有一个 `文风.md` 全写在里面

## 写什么

**写你对文风的具体要求。** 越具体越好，AI 能理解自然语言。

### ✅ 好的写法

```markdown
# 文风要求

- 对话要像真人说话，带口语感，不要书面语
- 动作描写要短促，一个动作一句话，不要堆形容词
- 每段不超过 3 句，留白让读者自己想
- 环境描写只在必要时出现，不要为了凑字数写风景
- 心理活动用行为暗示，不要直接说"他感到愤怒"
```

```markdown
# 对话风格

- 主角说话简短、带刺，经常用反问
- 女主说话温柔但有主见，不卑不亢
- 反派说话文雅但暗藏杀机
- 避免所有角色用同一个说话方式
```

```markdown
# 禁忌

- 不要写"他深吸一口气"——太俗
- 不要写"空气中弥漫着"——AI 味
- 不要写"命运的齿轮开始转动"——出戏
- 不要在战斗中插入大段回忆
```

### ❌ 不好的写法

```markdown
# 文风

写得好一点。
```
→ 太模糊，AI 不知道"好"是什么意思。

```markdown
# 文风

用鲁迅的风格写。
```
→ 太宽泛，AI 可能过度模仿导致不自然。应该具体说明要模仿的哪些特点。

```markdown
# 文风

第一段用 5 个字，第二段用 8 个字，第三段……
```
→ 过度微观，限制 AI 的创作空间。约束应该面向"效果"而非"字数"。

## 与系统约束的关系

系统有 5 层文风约束，你的提示词排在第二优先级：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1（最高） | 章级合同 | 每章的具体目标和禁止区域 |
| **2** | **你的提示词** | **你对文风的个性化要求** |
| 3 | 全局文风 | 系统从总纲推导的调性 |
| 4 | 写作技法 | BM25 检索匹配的技法库 |
| 5 | 审查维度 | reviewer agent 的 13 维度检查 |

如果你的提示词和系统约束冲突，以你的为准（但章级合同的 `forbidden_zones` 不可覆盖）。

## 常见场景

### 场景 1：我想让对话更自然

```markdown
# 对话

- 对话要像真人聊天，可以有打断、沉默、答非所问
- 不要每句话都完整，人说话经常只说半句
- "他这么说是因为……"这种解释性叙述全部删掉
- 对话要有潜台词，角色不会把心里话全说出来
```

### 场景 2：我想控制节奏

```markdown
# 节奏

- 每章开头 200 字内必须出现冲突或悬念
- 中段要有一次"呼吸"（短句、动作、场景切换）
- 章末留一个未解决的问题，不要"安全着陆"
- 紧张时用短句，舒缓时用长句
```

### 场景 3：我想避免 AI 味

```markdown
# 去 AI 味

- 禁止"缓缓""淡淡""微微"——这三个词每章最多出现 1 次
- 禁止"他终于明白了""由此可见"——不要总结句
- 禁止"首先……其次……最后"——不要列举
- 禁止每段结尾都用感叹号
- 情绪不要标签化（"他感到愤怒"），用行为展示（"他把杯子摔了"）
```

### 场景 4：我想写特定风格

```markdown
# 硬汉风格

- 短句为主，每句不超过 15 个字
- 少用形容词，多用动词
- 环境描写只写声音和气味，不写视觉
- 主角的内心独白用第一人称，其他用第三人称
- 对话不要引号外的解释
```

## 注意事项

1. **不要写太多**——500 字以内足够。写太多会挤占其他上下文的 token 预算
2. **不要写剧情**——提示词是"怎么写"，不是"写什么"。剧情交给大纲
3. **不要写设定**——世界观、力量体系等设定放在 `设定集/` 的对应文件里
4. **可以随时修改**——改了之后下一章就会生效，不需要重启任何服务
5. **可以分文件**——`对话.md`、`节奏.md`、`禁忌.md` 分开管理更清晰

## 与 Dashboard 的关系

Dashboard 的"文风约束"页面（`/style`）可以查看系统自动生成的约束（master_constraints、anti_patterns），但你的自定义提示词需要直接编辑文件。

这是设计意图：**系统推导的约束用 Dashboard 看，你自己的约束用文件管理。**
