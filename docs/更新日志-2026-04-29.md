## 四、可观测性系统

### 新增聚合方法（`observability.py`）

```python
from data_modules.observability import read_perf_timings, compute_stats, format_perf_report

# 读取 JSONL 数据
timings = read_perf_timings(project_root, tool_name="context_manager.build_context", limit=100)

# 按工具分组统计
stats = compute_stats(timings)
# 返回: {tool_name: {count, avg_ms, p50_ms, p95_ms, p99_ms, error_rate, last_run}}

# 格式化输出
print(format_perf_report(stats, fmt="text"))
```

### 新增 CLI 命令

```bash
# 性能总览
python .opencode/scripts/webnovel.py observability report

# 按工具过滤
python .opencode/scripts/webnovel.py observability report --tool context_manager.build_context --last 50

# JSON 格式输出
python .opencode/scripts/webnovel.py observability report --format json

# Token 消耗报告
python .opencode/scripts/webnovel.py observability token-report
```

### 性能分析

```bash

python .opencode/scripts/webnovel.py observability report

# 3. 查看 Token 消耗
python .opencode/scripts/webnovel.py observability token-report

# 4. 按工具深入分析
python .opencode/scripts/webnovel.py observability report --tool context_manager.build_context --format json
```

### 异常处理最佳实践

```python
from data_modules.exceptions import WebnovelError, ConfigError, StateManagerError

try:
    # 调用数据模块
    pass
except ConfigError as e:
    # 配置错误（参数、路径、JSON 格式）
    print(f"配置错误: {e}")
except StateManagerError as e:
    # 状态错误（文件锁、数据损坏）
    print(f"状态错误: {e}")
except WebnovelError as e:
    # 兜底：捕获所有自定义异常
    print(f"系统错误: {e}")
```
