<context>
此文件用于 Step 4 润色，目标是"修问题"而非"重写剧情"。

与 Step 2A 的职责边界：
- Step 2A：起草+风格（表达层，一次性产出）
- Step 4：问题修复（质量层），包括审查问题修复、Anti-AI 终检、毒点规避

若审查无 critical/high 问题，可跳过本步骤直接进入 Step 5。
</context>

<instructions>

## 1. 输入契约

```json
{
  "chapter_file": "正文/第0123章-章节标题.md",
  "overall_score": 82,
  "issues": [
    {"agent": "consistency-checker", "type": "POWER_CONFLICT", "severity": "critical", "location": "第6段", "suggestion": "境界越权"},
    {"agent": "ooc-checker", "type": "OOC", "severity": "high", "location": "第9段", "suggestion": "角色口吻偏离"}
  ]
}
```

## 2. 执行顺序

1. 修复 `critical`（必须，无法修复记录 deviation）
2. 修复 `high`（优先，无法修复记录 deviation）
3. 处理 `medium/low`（按收益择优）
4. 执行 Anti-AI 全文终检
5. 执行 No-Poison 毒点规避
6. 输出润色结果与 deviation

类型对应修复动作：
- OOC → 恢复角色话术/风险偏好/决策边界
- POWER_CONFLICT → 能力回落到合法境界，或补"获得路径+代价"
- TIMELINE_ISSUE → 补足时间流逝锚点
- LOCATION_ERROR → 补移动过程与空间锚点
- PACING_IMBALANCE → 增加推进事件或删冗余说明
- CONTINUITY_BREAK → 补衔接句与过渡动作

## 3. Anti-AI 检测（7层合并为3层核心 + 1层补充）

### 核心第1层：高风险词汇库（覆盖8类）

| 类别 | 关键词群 | 处理 |
|------|---------|------|
| 总结归纳 | 综合/总之/总而言之/由此可见/归根结底/简而言之/可以说/结论是/综上 | 默认替换 |
| 枚举模板 | 首先/其次/最后/第一/第二/第三/一方面/另一方面/再者/此外 | 重写句群 |
| 书面学术 | 某种程度/意义上/维度上/层面/体现为/构成/形成/实现/呈现/折射 | 换口语 |
| 逻辑滥用 | 因此/因而/然而/不过/但是/与此同时/对应/相应/从而/进而/于是/故而 | 删除 |
| 情绪直述 | 非常愤怒/五味杂陈/百感交集/情绪复杂/不由得感慨/内心一震 | 改行为句 |
| 动作套话 | 皱起眉头/叹了口气/深吸一口气/缓缓开口/沉声说道/嘴角上扬/眼神一凝/心头一跳 | 改具体动作 |
| 环境套话 | 空气仿佛凝固/气氛骤然紧张/四周一片寂静/死一般的寂静/时间仿佛静止 | 改感官描写 |
| 机械开场/收尾 | 故事要从/让视线转到/与此同时在另一边/命运的齿轮/新的篇章/未完待续 | 删除 |

### 核心第2层：句式反模板规则

- 禁止"首先/其次/最后"三段式（severity: high）
- 禁止连续3句同构句（severity: high）
- 禁止清单化叙事"一二三"（severity: high）
- 禁止同信息重复2-3次（severity: medium）
- 每3-5段至少一次句长变化
- 段尾禁止机械总结（severity: high）

### 核心第3层：对话/段落去AI化

- 禁止说明书式对话（severity: high）
- 对话必须有意图（试探/回避/施压/诱导）
- 对话后禁止追加解释（severity: medium）
- 全员同一反应模板 → severity: high
- 单句成段 25-45%，段长 20-100字
- 场景切换用动作/声音/位置，不用"第一幕"标签

### 补充层：形容词/四字套语/标点

- 连续2+形容词→重写；每300字"很/非常/极其"≤4次
- 每500字四字套语≤3个且不连续；成语需承担叙事功能
- 禁止连续`......`或`！！！`；省略号/感叹号每段最多1次

### 改写算法

1. 抽象情绪句 → 生理反应+当下意图+下一动作
2. 结论句 → 事实细节+代价/风险+决策
3. 连续说明句(≥3) → 对白/动作/反问混排
4. 连续同构句(≥3) → 至少1句改为短句或插入动作锚点
5. 对话整段解释背景 → 带意图的对抗式对白

## 4. No-Poison 毒点规避（5类）

1. 降智推进：角色忽略常识推进剧情 → 补动机/代价
2. 强行误会：可一句话说清却拖延 → 补阻力
3. 圣母无代价：无边界原谅 → 补代价
4. 工具人配角：仅功能节点出现 → 补独立动机
5. 双标裁决：同行为不同评价标准 → 补叙事解释

命中任一 → 必须补"动机/阻力/代价"至少两项。

## 5. 润色红线

- 不改剧情走向（大纲即法律）
- 不改设定物理边界（设定即物理）
- 不删除关键伏笔
- 不强行改写角色关系基线

## 6. 输出格式

必须输出：
1. 润色后正文
2. 修复摘要：

```text
[润色报告]
- 严重问题已修复: N 处
- 高优先级已修复: N 处
- Anti-AI 改写: N 处
- anti_ai_force_check: pass/fail
- 毒点风险: pass/fail
- 偏离记录: {位置}: {原因}
```

### 通过阈值

- 全文无未处理的三段式枚举模板句
- 命中高风险表达均已改写或记录 deviation
- 至少3处"抽象句→行为句"改写
- 章末存在可感知的下一步压力或未闭合问题
- `anti_ai_force_check=pass`（fail 则不得进入 Step 5）
