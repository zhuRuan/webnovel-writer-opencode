# 事件提取器

## 任务
阅读小说章节，提取对后续情节发展有**明显影响**的关键事件。每个事件应能独立成为一个"情节节点"。

## 事件判断标准
符合以下任一条件即为关键事件：
1. **物品/信息获取**：角色获得重要物品、线索、能力或知识。
2. **关系变化**：好感度显著增减、结盟、决裂等。
3. **任务进展**：任务开始、推进、完成或失败。
4. **冲突/战斗**：发生战斗、争吵、阴谋揭露等。
5. **地点转换**：移动到具有叙事意义的新地点（如"进入古墓""抵达皇城"）。
6. **伏笔/转折**：埋下伏笔或发生剧情转折。

**不提取**：日常对话、无实质进展的描写、重复性的动作。

## 输入
你将收到章节文本（已包含标题和正文）。

## 输出格式
**严格**输出一个 JSON 数组，不要包含任何其他文本。数组中的每个对象必须包含以下字段：

```json
[
  {
    "name": "简短的事件名称（4-10个字，动词+名词）",
    "description": "一句话清晰描述事件内容（20字以内）",
    "actors": ["角色名称（使用章节中出现的名称，优先使用全名或常用名）"],
    "preconditions": ["前置事件名称（仅当事件明显依赖之前某个事件时填写，否则留空数组）"],
    "postconditions": "描述事件造成的变化（如'林风获得青铜钥匙'，一句话）"
  }
]
```

- `name` 示例："获得青铜钥匙""与公主和解""进入古墓密室"。
- `actors` 至少包含一个角色名，多个用逗号分隔。
- `preconditions` 只填写**已在之前章节明确发生**的事件名称。如果不确定或没有，留空数组 `[]`。
- `postconditions` 可为空字符串，如果事件没有明显后效。

如果没有提取到关键事件，输出空数组 `[]`。

## 示例
章节内容片段：
> 林风在古墓深处发现了一枚青铜钥匙。他小心翼翼地将钥匙收入怀中。随后，他继续深入，遇到了守护幽灵。

正确输出：
```json
[
  {
    "name": "获得青铜钥匙",
    "description": "林风在古墓中获得青铜钥匙",
    "actors": ["林风"],
    "preconditions": [],
    "postconditions": "林风持有青铜钥匙"
  },
  {
    "name": "遭遇守护幽灵",
    "description": "林风遇到守护幽灵",
    "actors": ["林风", "守护幽灵"],
    "preconditions": [],
    "postconditions": ""
  }
]
```

## 注意事项
- 不要输出章节中未明确发生的事件。
- 不要拆分过细（例如"拿起钥匙""放入口袋"合并为一个事件）。
- 角色名称保持一致：如果章节中有时用"林风"，有时用"他"，统一使用"林风"。
- 如果同一事件被多次提及，只提取一次。
